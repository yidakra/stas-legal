# Установка и запуск dir2mcp сервера
## 0. Если вы ставите заново (полная очистка)

Если раньше уже что-то ставили — удалите старую версию тула. Если ставите
впервые, **пропустите этот шаг** и идите к шагу 1.

```bash
# удалить старые версии тула и отключить старый источник
brew uninstall dir2mcp-full dir2mcp
brew untap dirstral/tap
```

> Папку `stas-legal` с документами скачивают **все** на шаге 4 — и при
> первой установке, и при переустановке. Отдельно здесь её скачивать не
> нужно.

---

## 1. Homebrew (если его ещё нет)

Если не уверены — выполните, команда безопасна. Если Homebrew уже есть,
переходите к шагу 2.

```bash
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

---

## 2. Bun (нужен для подключения к Claude)

Ставим сразу, чтобы потом не споткнуться на шаге подключения.

```bash
brew tap oven-sh/bun
brew install bun
```

---

## 3. Установить dir2mcp (версия с Docling)

```bash
brew tap dirstral/tap
brew trust dirstral/tap
brew install dirstral/tap/dir2mcp-full
```

`dir2mcp-full` — полная версия с Docling: лучше разбирает структуру
юридических PDF (разделы, таблицы, схемы).

> Строка `brew trust dirstral/tap` нужна новым версиям Homebrew: без неё
> установка может прерваться с ошибкой `Refusing to load formula … from
> untrusted tap`. Команда разово помечает наш источник как доверенный.
> Если ваш Homebrew её не знает (`Unknown command: trust`) — просто
> пропустите эту строку, она не требуется.

---

## 4. Скачать папку stas-legal и зайти в неё

Сначала скачайте папку с документами с GitHub (этот шаг выполняют **все** —
и при первой установке, и при переустановке):

```bash
cd ~/Downloads
git clone https://github.com/yidakra/stas-legal.git
```

> Нет `git`? Откройте https://github.com/yidakra/stas-legal →
> зелёная кнопка **Code** → **Download ZIP**, распакуйте в `~/Downloads`
> (должна получиться папка `~/Downloads/stas-legal`).
>
> Папка уже скачана с прошлого раза? Пропустите клонирование и сразу
> выполните команду `cd` ниже.

Теперь зайдите в эту папку:

```bash
cd ~/Downloads/stas-legal
```

> Совет: можно написать `cd ` (с пробелом в конце) и перетащить папку из
> Finder прямо в окно терминала, затем Enter.

---

## 5. Настройка через мастер (ключи + профиль)

**Ваши ключи** (понадобятся в мастере ниже — скопируйте нужный и вставьте,
когда мастер спросит):

```
Mistral API key:  СЮДА_КЛЮЧ_MISTRAL
Cohere API key:   СЮДА_КЛЮЧ_COHERE
```

Запустите пошаговый мастер — он спросит ключи и тип корпуса и сам создаст
все нужные файлы настроек:

```bash
dir2mcp config init
```

Мастер проведёт по экранам (между полями — клавиша **Enter**; в списках
выбор стрелками **↑/↓**, подтверждение **Enter**):

1. **Mistral API key** — обязательный ключ (распознавание PDF и ответы).
   Вставьте `Mistral API key` из блока «Ваши ключи» выше (ввод скрыт — это
   нормально), нажмите Enter.
2. **Cohere API key** — рекомендуется. Добавляет переранжирование: после
   поиска ещё раз пересортировывает найденные фрагменты по релевантности,
   поэтому в ответ попадают самые точные разделы — для юридических вопросов
   заметно точнее. Вставьте `Cohere API key` из блока выше (или оставьте
   пустым и нажмите Enter, чтобы пропустить).
3. **Configure optional chat / audio providers?** — выберите **No**
   (другие провайдеры не нужны).
4. **Where to store credentials?** — выберите **`.env.local` file**
   (надёжно работает с фоновым сервером). Вариант «OS keychain» тоже есть
   — он безопаснее (ключи в системном хранилище), но для фонового сервера
   проще и стабильнее файл.
5. **Corpus profile** — выберите **Legal / citations (strict grounding)**.
   Это включает режим: отвечать строго по документам, указывать акт +
   раздел, брать больше контекста на вопрос.
6. **Save these settings?** — **Save**.

Готово — ключи и профиль сохранены. (Тот же мастер запускается
автоматически при первом `dir2mcp up`, если ключ ещё не задан.)

> Ввод ключей скрыт (точки/пусто) — так и должно быть. Если ошиблись,
> просто запустите `dir2mcp config init` ещё раз.

---

## 6. Запустить сервер

```bash
dir2mcp up
```

При первом запуске тул проиндексирует все документы (распознавание PDF
через Mistral) — несколько минут. Дождитесь сообщения о готовности;
дальше сервер работает в фоне.

Проверить состояние в любой момент:

```bash
dir2mcp status
```

---

## 7. Подключить к Claude Desktop

```bash
dir2mcp install claude
```

После этого **полностью перезапустите приложение Claude** (закрыть и
открыть заново), чтобы оно увидело новый инструмент.

> Ошибка `could not find bunx or npx in PATH`? Значит пропущен шаг 2
> (bun). Выполните шаг 2 и повторите `dir2mcp install claude`.

---

## 8. Проверка — спросите в приложении Claude

Откройте Claude Desktop и задайте вопросы (фраза `Use dir2mcp` подсказывает
Claude использовать наш инструмент):

1. `What powers does the BVI Financial Investigation Agency have? Cite sections. Use dir2mcp`
2. `What entities are subject to BVI economic substance requirements? Use dir2mcp`
3. `Under what circumstances is a financial institution required to file a suspicious transaction report in the BVI? Include the section number. Use dir2mcp`

В ответах должны быть ссылки на конкретные акты и разделы.

---

## Полезные команды

```bash
dir2mcp status            # состояние сервера и статистика корпуса
dir2mcp down              # остановить сервер для этой папки
dir2mcp config init       # перезапустить мастер (ключи / профиль)
dir2mcp config secrets    # показать, какие ключи заданы (без значений)
dir2mcp uninstall claude  # убрать сервер из конфигурации Claude
dir2mcp reindex           # принудительно переиндексировать (обычно не нужно)
```

> Убрать сервер из Claude можно и в приложении: Settings → Developer.

---

## Если что-то пошло не так

- **Claude не видит инструмент** — перезапустили ли приложение Claude
  после `dir2mcp install claude`? Полностью закройте и откройте заново.
- **Ошибка про ключ / пустые ответы** — запустите `dir2mcp config init`
  заново и убедитесь, что вставили настоящий **Mistral**-ключ. После этого
  перезапустите сервер: `dir2mcp down`, затем `dir2mcp up`. Проверить, какие
  ключи заданы: `dir2mcp config secrets` (значения не показываются).
- **Хочу поменять/добавить ключ (например Cohere)** — просто запустите
  `dir2mcp config init` ещё раз и пройдите мастер заново.
- **`command not found: dir2mcp`** — установка (шаг 3) не прошла; повторите.
- **`Refusing to load formula … from untrusted tap`** — выполните
  `brew trust dirstral/tap` и повторите установку (шаг 3).
- **После обновления `dir2mcp version` показывает старую версию** —
  выполните `hash -r` или откройте новое окно терминала.
- **Claude отвечает «из логов» — ссылается на `.log`, `.yaml`, `status.json`
  или `SETUP.md` вместо законов, либо не может открыть PDF** — значит в папку
  попали служебные файлы и индекс «засорился». Чаще всего это архив диагностики
  `dir2mcp-support-*.tar.gz`, оставленный в папке. Удалите лишнее и
  переиндексируйте:
  ```bash
  cd <папка stas-legal>
  rm -f dir2mcp-support-*.tar.gz
  dir2mcp reindex
  ```
  В папке должны лежать только сами документы (PDF и т. п.), без архивов и
  служебных файлов.
- Собрать диагностику для разработчика — **сохраняйте архив ВНЕ папки
  stas-legal**, иначе он попадёт в индекс и ответы начнут ссылаться на
  служебные логи:
  ```bash
  dir2mcp support-bundle --output ~/Downloads/dir2mcp-support.tar.gz
  ```
